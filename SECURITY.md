# Security Policy

## Reporting a Vulnerability

The Otim team takes security vulnerabilities seriously. We appreciate your efforts to responsibly disclose your findings.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via email to: security@otim.com

Please include the following information in your report:

- Type of vulnerability
- Full paths of source file(s) related to the vulnerability
- Location of the affected source code (tag/branch/commit or direct URL)
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the vulnerability

## Response Timeline

- We will acknowledge receipt of your report within 48 hours
- We will provide a more detailed response within 7 days
- We will work with you to understand and resolve the issue promptly

## Disclosure Policy

- We request that you give us reasonable time to address the issue before public disclosure
- We will coordinate with you on the disclosure timeline
- We will credit you for the discovery (unless you prefer to remain anonymous)

## Scope

This security policy applies to the `otim-go-sdk` repository and its official releases.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

We recommend always using the latest version of the SDK.
